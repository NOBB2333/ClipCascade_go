# ClipCascade 改进与重构操作全记录

本文件详细记录了针对 ClipCascade 项目进行的一系列优化、修复及架构重构的操作步骤。

## 阶段 1：服务端启动与日志可视化
- **操作描述**：优化服务端启动信息，使其更易于在局域网定位。
- **具体步骤**：
    1. 修改 `server/main.go`，增加 `getLocalIPs` 函数，使其在启动时列出所有物理/虚拟网卡的 IP（Vite 风格）。
    2. 修改 `ws_hub.go` 和 `p2p_signaling.go`，在客户端连接/断开时记录远程 IP 地址。
    3. 调整 `pkg/constants/constants.go`，将默认消息限制提升至 20 MiB。

## 阶段 2：初步稳定性修复 (Windows)
- **操作描述**：解决 Windows 端的拦截误报与断连问题。
- **具体步骤**：
    1. 在 `monitor_windows.go` 中引入初步的 `os.Stat` 校验，防止将普通文本路径误认为大文件。
    2. 在 `stomp_client.go` 中加入 30s 的 Ping 心跳发送逻辑。

## 阶段 3：服务端自动发现增强
- **操作描述**：实现“零配置”连接。
- **具体步骤**：
    1. 修改 `application.go`，当手动配置的服务器连接失败时，自动触发 mDNS 全局搜索。
    2. 在重连循环中增加探测机制，应对服务器 IP 变动。

## 阶段 4：深度发现优化 (多 IP 容错)
- **操作描述**：解决多网卡引发的发现卡死问题。
- **具体步骤**：
    1. 重构 `discoverServer` 使其返回所有候选 IP 列表。
    2. 优化 `connect` 逻辑，支持按发现顺序自动遍历尝试所有 IP，直到登录成功。
    3. 增加 Windows 路径启发式前缀校验（盘符/UNC 检查）。

## 阶段 5：全平台发送端过滤
- **操作描述**：彻底消除跨平台误报。
- **具体步骤**：
    1. 将磁盘存在性校验 (`os.Stat`) 同时也应用到 macOS 和 Linux 的发送端识别中。
    2. 确保如果文件在本地不存在，则绝不作为 `file_stub` 发送。

## 阶段 6：【核心】架构重构与稳定性加固
- **操作描述**：从“内容猜测”进化为“事件驱动”，并解决 i/o timeout。
- **具体步骤**：
    1. **统一事件循环**：重构 `monitor.go`，由轮询内容改为监听系统变动计数器 (`Change Count` / `SequenceNumber`)。
    2. **严格优先级识别**：在 `handleSystemChange` 中确立“物理文件 > 图像 > 文本”的单次原子识别顺序。只有系统标记是文件且磁盘存在时，才发文件。
    3. **通信加固**：
        - 实现 `PongHandler`，只有收到服务端回响才重置读取截止时间，彻底终结了 `i/o timeout` 触发的频繁重连。
        - 增加了 `wsMu` 互斥锁，确保 P2P 信令层的并发安全。
    4. **交互增强**：增加 `ui.Notify` 调用，为接收文件和拦截操作提供桌面视觉反馈。
